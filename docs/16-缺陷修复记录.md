## BUG-001 · Codex resume 会话 id 撞车 → 右侧详情永远错乱

- **发现**：2026-06-02（阶段 7 之后）。报告者：用户。
- **严重度**：高（部分 Codex 会话无法正确查看/导出）。
- **状态**：✅ 已修复并验证。

### 现象
左栏同一天有两份标题相同的 Codex 对话（如「你现在扮演一名资深 AI 产品架构师…」，分别 20 轮 / 9 轮）。无论点哪一个，右侧**始终只显示 9 轮**那份；点 20 轮没有任何变化。

### 根因
Codex 会话 `sessionId` 此前取自首行 `session_meta.payload.id`（`server/src/parser/codex.ts`）。但该 id 是**「会话线程 id」，会在 resume/续接时被多个 rollout 文件共用**，并非每个文件唯一：

| 会话 | rollout 文件名（各自 uuid） | `session_meta.id` | 旧算法得到的 id |
|---|---|---|---|
| 9 轮 (13:07) | `…T13-07-15-`**`019e7221`**`-…` | `019e7221` | `codex:019e7221` |
| 20 轮 (13:47) | `…T13-47-19-`**`019e7245`**`-…` | `019e7221` | `codex:019e7221` ← 撞车 |

`id`（`${source}:${sessionId}`）是整条链路主键：前端列表 `key={s.id}`、选中高亮 `s.id===selectedId`、`useSession(selectedId)`；后端 `resolveSessionPath` 按「文件名包含 sessionId」反查。两会话 id 相同 ⇒ 点哪个 `selectedId` 都一样 ⇒ 反查只命中文件名里含 `019e7221` 的 9 轮文件（20 轮文件名是 `019e7245`）⇒ 永远返回 9 轮。

**证据**：`node src/cli.ts scan --source codex --json` 全库 96 个 Codex 会话里有 **10 组撞 id**（resume/fork 的会话线程）。

### 修复
把 Codex 的 `sessionId` 改为 **rollout 文件名里的 uuid**（每个文件唯一），不再用 `session_meta.id`。Claude 不动（其 uuid 本就每文件唯一）。

- 实现：`server/src/parser/index.ts` `parseSessionFile` 对 codex 用 `codexSessionIdFromPath(filePath)` 覆盖 `sessionId`——正则取文件名末尾标准 uuid，非常规文件名兜底用整段 basename。
- `resolveSessionPath`（文件名包含 + 内部 `meta.sessionId` 核对）天然兼容，反查更精确。

**为什么用「文件名 uuid」而非整段文件名**：原始（未 resume）会话的*文件名 uuid 恰等于其 `session_meta.id`*（线程 id 诞生自首个 rollout）。所以原始会话「新 id == 旧 id」，现有 manifest 徽章不丢；只有被 resume 的会话拿到各自唯一的新 id，撞车消除。

### 改动文件
| 文件 | 改动 |
|---|---|
| `server/src/parser/index.ts` | codex `sessionId` 改用文件名 uuid（新增 `codexSessionIdFromPath` + `UUID_RE`） |
| `server/test/parser.test.ts` | 新增回归：同 `session_meta.id`、异文件名 → 两个不同会话 id |
| `server/test/scanner.test.ts` | codex id 断言改为文件名 uuid 派生 |
| `server/test/fixtures/scan/codex/2026/06/01/` | fixture 重命名为带真实 uuid 的文件名（`…-019e0000-…c1.jsonl`） |
| `docs/03 §D` | 修正 Codex `sessionId` 来源说明 |

前端**无改动**——后端 id 唯一后，列表/选中/详情自然正确。

### 测试与验证
- `node --test`：**63 全绿**（原 62 + 1 回归）。
- 真实数据复查：`scan --source codex` → 96 会话、**重复 id 组数 0、唯一 id 96**；原撞车的 20 轮 (`019e7245`) 与 9 轮 (`019e7221`) 现为不同 id。
- 类型检查（借 web tsc）exit 0。

### 迁移影响（manifest）
- `id` 算法变更只影响**被 resume 的** Codex 会话；原始会话 id 不变。
- 现有 `server/data/manifest.json` 唯一的 codex 记录 `codex:019e68d7-…`（MCP 原理）经核对 `019e68d7` 正是某原始会话的文件名 uuid → **id 不变，已导出记录完整保留**。
- 其余撞车会话此前基本无法访问、也无导出记录，无迁移损失。
