## 1. 设计原则

- **provider 无关**：同一接口适配 OpenAI 兼容 / Anthropic / 本地 Ollama，靠配置切换。
- **可降级**：未配置或调用失败时，自动退回"原样拼接导出"（即关闭 AI 整理的行为），不阻断主流程。
- **隔离**：AI 逻辑全部收敛在 `server/ai/` 模块，导出主流程只调一个函数。

## 2. 配置 Schema

存于 `server/data/config.json`（key 不回传前端明文）。
> 阶段 2 补注：实际 `config.json` 在 `ai` 之外**新增顶层 `exportDir`**（默认 = `paths.DEFAULT_EXPORT_DIR`），
> 供 `POST /api/export` 取导出目录（`docs/02 §4` 的 export 入参不含 `dir`，`docs/05 §1` 约定存后端配置）。
> `GET /api/config` 返回整体并对 `ai.apiKey` 脱敏。详见 `docs/09 §5`。

```jsonc
{
  "exportDir": "E:\\...\\02_AI对话记录",   // 阶段 2 新增；默认 DEFAULT_EXPORT_DIR
  "ai": {
    "enabled": false,
    "provider": "openai-compatible",   // openai-compatible | anthropic | ollama
    "baseUrl": "https://api.openai.com/v1",  // 各家填各家；Ollama 如 http://localhost:11434
    "apiKey": "",                       // Ollama 可空
    "model": "gpt-4o-mini",            // 或 claude-*, 或本地模型名
    "skill": "default",                // 整理用的 skill / 提示模板名
    "temperature": 0.3,
    "timeoutMs": 60000
  }
}
```

各 provider 的差异（后续实现时填）：
| provider | 接口 | 鉴权 | 备注 |
|----------|------|------|------|
| openai-compatible | `POST {baseUrl}/chat/completions` | `Authorization: Bearer` | 覆盖 OpenAI / DeepSeek / 通义 / Kimi / 智谱 等 |
| anthropic | `POST {baseUrl}/v1/messages` | `x-api-key` + `anthropic-version` | Claude 官方 |
| ollama | `POST {baseUrl}/api/chat` | 无 | 本地、不联网 |

## 3. 接口契约

> ⛔ 以下为阶段 5 的**单篇**契约，阶段 7 已取代为多篇 `TidyResult{ usedAi, notes:[{filename,content}], noNotes?, degraded? }`，见 [docs/15 §3](15-阶段7-AI真实接入-实现计划.md)。保留此节仅作历史。

导出主流程只认这一个函数：

```ts
// server/ai/tidy.ts
interface TidyInput {
  title: string;
  source: 'claude' | 'codex';
  blocks: { id: string; role: 'question' | 'answer'; text: string }[];
}
interface TidyResult {
  markdown: string;          // 整理后的正文（不含 front matter，front matter 由 exporter 加）
  suggestedFilename?: string; // 用于"AI 介入后自动命名"
  usedAi: boolean;            // 是否真的走了模型（false=降级为原样拼接）
}

export async function tidy(input: TidyInput, cfg: AiConfig): Promise<TidyResult>;
```

导出流程伪代码：
```text
blocks = 选中的块
if cfg.ai.enabled:
    result = await tidy({title, source, blocks}, cfg.ai)
else:
    result = { markdown: 原样拼接(blocks), usedAi: false }
md = front_matter(meta, result) + result.markdown
写文件(filename 或 result.suggestedFilename)
manifest 记录(aiTidy = result.usedAi)
```

## 4. 本期占位实现

```ts
export async function tidy(input: TidyInput, cfg: AiConfig): Promise<TidyResult> {
  // TODO(后续迭代): 按 cfg.provider 调真实模型，套用 cfg.skill 的提示词整理 input.blocks
  // 本期：原样拼接，标记未使用 AI
  const markdown = input.blocks
    .map(b => `## ${b.role === 'question' ? '提问' : '回复'}\n\n${b.text}`)
    .join('\n\n');
  return { markdown, usedAi: false };
}
```
