# 02 · 架构设计

## 1. 总览

```
┌────────────────────────────────────────────────────────────┐
│                     浏览器（前端 web/）                       │
│   封面页  →  两栏 App（左：分类/日期/列表  右：阅读/勾选/导出）   │
│   React + Vite + Tailwind + shadcn/ui ·  晨雾 Sage Mist 风格   │
└───────────────▲───────────────────────────┬──────────────────┘
                │ HTTP (localhost)           │ fetch JSON
                │                            ▼
┌───────────────┴───────────────────────────────────────────────┐
│                     本地服务（后端 server/）                     │
│  Node.js · 只读源 jsonl，写导出 md + 维护已导出索引              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌───────┐ │
│  │ scanner  │ │ parser   │ │ exporter │ │ manifest │ │ ai *  │ │
│  │ 扫描会话  │ │ 解析分轮  │ │ 生成md    │ │ 已导出索引│ │ 占位  │ │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘ └────┬─────┘ └───┬───┘ │
└───────┼────────────┼────────────┼────────────┼───────────┼─────┘
        │ 读          │ 读          │ 写          │ 读/写      │（本期空实现）
        ▼            ▼            ▼            ▼           ▼
  ~/.claude/projects/**.jsonl              导出目录/*.md   配置的大模型 API
  ~/.codex/sessions/**/*.jsonl             manifest.json   (OpenAI兼容/Anthropic/Ollama)
```

## 2. 模块职责（后端）

| 模块 | 职责 | 关键产物 |
|------|------|----------|
| **scanner** | 扫描两个数据源目录，列出会话（不解析正文），返回 id/来源/时间/标题/轮数 | 会话清单 |
| **parser** | 解析单个会话 jsonl → 过滤噪音 → 按"轮"切分 → 输出干净结构 | `Round[]` |
| **exporter** | 把选中的块拼成 Markdown（含 YAML front matter），写入导出目录 | `.md` 文件 |
| **manifest** | 维护"已导出"索引（哪个会话的哪些块、何时、导出到哪个文件） | `manifest.json` |
| **ai（占位）** | 第 5 点接口：`tidy(blocks, config)`，本期返回原样拼接 + TODO | 见 `docs/06` |

## 3. 数据流（典型操作）

1. 用户点 **Claude** 分类 → 前端 `GET /api/sessions?source=claude&range=7d`
   → scanner 返回会话清单（含 `exported` 状态，来自 manifest）。
2. 用户点某条标题 → `GET /api/sessions/:id`
   → parser 解析该会话 → 返回 `Round[]`（每轮含 question / answer 文本 + 已导出标记）。
3. 用户勾选若干块，填文件名，点导出 → `POST /api/export`
   → exporter 写 md → manifest 记录 → 返回结果路径。
4.（可选）打开"AI 介入整理" → export 时走 ai 模块（本期占位）。

## 4. HTTP 接口约定（草案）

| Method | Path | 入参 | 出参 |
|--------|------|------|------|
| GET | `/api/sources` | — | `{claude:{count}, codex:{count}}` |
| GET | `/api/sessions` | `source`, `range`(7d\|30d\|all) | `Session[]`（见下） |
| GET | `/api/sessions/:id` | `id`（含 source 前缀） | `{meta, rounds: Round[]}` |
| POST | `/api/export` | `{id, blockIds[], filename, aiTidy}` | `{ok, path}` |
| GET | `/api/config` | — | AI 配置（脱敏） |
| PUT | `/api/config` | AI 配置 | `{ok}` |

### 数据结构（建议 TypeScript）

```ts
type Source = 'claude' | 'codex';

interface Session {
  id: string;          // `${source}:${sessionId}`，全局唯一
  source: Source;
  title: string;       // 首条人类提问（截断）
  startedAt: string;   // ISO 时间
  day: string;         // YYYY-MM-DD，用于日期分组
  project?: string;    // 来自记录里的 cwd
  turnCount: number;   // 人类提问的轮数
  exported: 'none' | 'partial' | 'full';
}

interface Round {
  index: number;       // 第几轮，从 1 开始
  question: Block;     // 人类提问
  answer: Block;       // AI 文字回复（多段合并）
}

interface Block {
  id: string;          // `r${index}-q` | `r${index}-a`
  text: string;
  exported: boolean;   // 该块是否已导出（来自 manifest）
}
```

> 解析与过滤的**具体规则**见 `docs/03`，导出格式与 manifest 见 `docs/05`。

## 5. 前端结构

```
web/
├─ src/
│  ├─ pages/Cover.tsx        # 封面页（先展示）
│  ├─ pages/App.tsx          # 两栏主界面
│  ├─ components/
│  │  ├─ SourceTabs.tsx      # Codex / Claude 分类
│  │  ├─ RangeFilter.tsx     # 7d / 30d / all
│  │  ├─ SessionList.tsx     # 按日期分组的列表 + 已导出徽章
│  │  ├─ ConversationReader.tsx  # 右侧阅读 + 按块勾选
│  │  ├─ ExportBar.tsx       # 文件名 + AI开关 + 导出按钮
│  │  └─ ui/                 # shadcn 组件
│  ├─ lib/api.ts             # 调后端
│  └─ styles/theme.css       # 晨雾配色 token（见 docs/04）
```

- **路由**：`/`（封面）→ 点"进入"→ `/app`。
- **状态**：轻量即可（fetch + 局部 state 或 React Query）。
- **懒加载**：分类未点击不请求列表；会话详情点击才请求。

## 6. 运行方式（阶段 6 已落地，详见 `docs/13`）

- **开发**：`web` 跑 Vite dev（5173），`server` 跑 Node API（8787），Vite 代理 `/api` 到后端。
- **日常用**：`server` 同时托管 `web/dist` + API（同源，免 CORS）；项目根 `start.cmd` 缺 `web/dist` 才构建 → 起服 → 自动开浏览器。
- 环境覆盖：`AIDA_PORT` / `AIDA_STATIC_DIR` / `AIDA_EXPORT_DIR` / `AIDA_MANIFEST` / `AIDA_CONFIG`。

## 7. 关键约束

- **只读源**：scanner/parser 绝不写 `.claude`/`.codex`。
- **路径来自记录内部**：会话所属项目（cwd）从 jsonl 记录里的 `cwd`/`session_meta` 读取，**不要**靠反解目录名（编码有歧义，见 `docs/03`）。
- **格式容错**：遇到未知 `type`/`payload.type` 跳过而非报错；上游格式可能演进。
