# 01 · 需求规格

## 1. 目标

把用户在 **Claude Code** 和 **Codex CLI**（Windows 桌面端）里与 AI 的对话记录，手动触发地、有选择地导出为 Markdown 文档，存到指定文件夹（默认 Obsidian 仓库）。

## 2. 用户与场景

- **用户**：本机单人使用（开发者本人）。
- **场景**：聊完一段有价值的对话后，想把其中**某几轮**的提问/回复挑出来，整理成笔记沉淀到 Obsidian；不想要整段、也不想手动复制粘贴和清洗噪音。

## 3. 功能需求

### FR-1 分类与懒加载
- 顶部有 **Codex / Claude** 两个分类入口。
- **点击分类后才加载**对应的会话列表（避免一次性加载上百个会话）。

### FR-2 按日期浏览 + 完整对话阅读
- 列表按**日期分组**（精确到某天），每组下列出当天的对话**标题**。
- 标题取该会话的**首条人类提问**（截断显示）。
- 点击标题 → 右侧展示该会话**完整对话内容**：
  - **只保留**人类提问 + AI 的文字回复。
  - **过滤掉**工具调用、工具结果、系统/环境上下文、文件快照等噪音。
- 右侧内容可**下拉滚动**，方便用户读全后再决定勾选什么。

### FR-3 时间范围筛选
- 可切换展示范围：**近 7 天 / 近 1 个月 / 全部**。

### FR-4 按块勾选导出
- 对话以"轮"为单位展示：第 N 轮提问、第 N 轮回复，**每块独立可勾选**。
- 例：可只勾"第 1 轮提问 + 第 1 轮回复 + 第 3 轮回复"。
- 勾选后一键导出为一个 `.md` 文件。

### FR-5 已导出提醒
- 界面需标记**哪些标题/对话已经导出过**：
  - 会话标题旁显示徽章（已导出 / 部分导出）。
  - 进入会话后，已导出的具体块也要有标注。

### FR-6 AI 介入整理开关（本期仅占位）
- 一个开关："是否 AI 介入整理"。
- 开启时：调用**配置的大模型 API**，按 **skill** 对勾选内容进行整理后存为 md。
- **本期只做接口/占位**，不接真实模型；接口需兼容 OpenAI 兼容 / Anthropic / 本地 Ollama。
- 关闭时（默认）：直接把勾选块拼成 md 导出。

### FR-7 封面 + 美观界面
- 启动后**先展示封面页**，而非直接进操作界面。
- 视觉风格：**晨雾 Sage Mist**（安静、舒适、享受；柔和自然色 + 留白 + 鼠标视差微动效，不堆科技感）。

## 4. 导出规则

- **导出目录默认**：`E:\Work2\Obsidians_database\00_资源库\02_AI对话记录`
  - AI 介入整理后的笔记也默认存这里。
  - 目录可在界面里随时更改。
- **文件命名**：
  - 手动模式：用户手动填写文件名。
  - AI 介入开启后：自动命名（见 `docs/06`）。

## 5. 非功能需求

- **本地优先**：纯本地读写，默认不联网（AI 整理开启时才按配置访问对应 API）。
- **只读源数据**：绝不修改 `.claude` / `.codex` 下的原始 jsonl。
- **零破坏**：导出是新增文件；已导出索引独立维护。
- **性能**：本机当前约 134 个会话（Claude 44 + Codex 90），列表懒加载、详情按需解析即可流畅。
- **可维护**：所有格式假设、解析规则写进 `docs/03`，便于上游格式变动时定位修改。

## 6. 范围之外（本期不做）

- 不做 Claude Desktop 纯聊天版 / ChatGPT Desktop（数据在服务器端，本地无完整记录）。
- 不做多用户、不做云同步、不做账号体系。
- 第 5 点 AI 整理的**真实模型调用**留到后续迭代。
