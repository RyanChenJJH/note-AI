## 1. 本期目标与范围

**目标**：把阶段 3 的 `/app` 占位壳替换成**可用的两栏工作界面**——左侧选会话、右侧读对话并按块勾选、底部填名导出，跑通完整链路：
`选源 → 选范围 → 列表 → 选会话 → 读分轮 → 勾块 → 命名 → 导出 → 徽章/块状态回写`。

**范围内**（= `docs/07` 阶段 4）：
- ✅ 左栏：Codex/Claude 分类 Tab（**点击才加载**该源列表）+ 时间范围筛选（7d/30d/all）。
- ✅ 左栏：按日期分组、可折叠列表；每条标题带已导出徽章（● full / ◐ partial / 无）。
- ✅ 右栏：完整对话阅读区，按轮分块、可滚动；每块独立 checkbox + 块级已导出标注。
- ✅ ExportBar：文件名输入 + AI 整理开关（占位，可切但提示降级）+ 「导出为 .md」按钮 → `POST /api/export`。
- ✅ 导出成功后：刷新该会话块级 `exported` 与列表徽章；提示导出路径。
- ✅ 微动效：列表入场 stagger、悬停轻浮、选中态青色描边；reduced-motion/focus/对比度过检。
- ✅ 思考过程开关（阅读区头部，调 `?thinking=`，可选默认关）。

**范围外**：
- ❌ AI 真实整理（开关只传 `aiTidy`，本期后端仍原样导出）→ **阶段 5**。
- ❌ 配置界面（导出目录/AI 表单）→ 配置接口已就绪，UI 留后续；本期导出目录用后端默认。
- ❌ server 托管构建产物 + 一键启动 → **阶段 6**。
- ❌ 列表虚拟化、暗色模式 → 后续迭代。

## 3. 状态与数据流

**页面级 state（AppShell 持有，向下传 props）**：
```ts
source: 'claude' | 'codex'        // 当前分类（默认 claude）
range: '7d' | '30d' | 'all'       // 默认 30d
selectedId: string | null         // 当前会话 id
selected: Set<string>             // 勾选的 blockId 集合（切会话时清空）
filename: string                  // 导出文件名（空则后端用标题前 20 字）
aiTidy: boolean                   // AI 整理开关（默认 false）
thinking: boolean                 // 是否显示思考过程（默认 false）
```

**懒加载缓存**（`useSessions`/`useSession` 内用 `useRef(Map)`）：
- `useSessions(source, range)`：key=`${source}|${range}`；首次进入或换 key 才请求；**未点的源不请求**。
- `useSession(id, thinking)`：key=`${id}|${thinking}`；选中会话/切 thinking 才请求。
- 导出后通过 `refetch()` 让对应 key 失效并重拉（拿到回写后的 `exported`）。

**典型链路**：
1. 点 Claude Tab → `useSessions('claude','30d')` 拉 `GET /api/sessions?source=claude&range=30d` → `group.ts` 按 day 分组渲染。
2. 点某条 → `useSession('claude:xxx', false)` 拉 `GET /api/sessions/:id` → 右栏分轮。
3. 勾若干块（toggle blockId 进/出 `selected`）→ 填名 → 点导出。
4. `postExport({id, blockIds:[...selected], filename, aiTidy})` → `{ok, path}` → toast/提示路径 → `refetch` 当前会话 + 当前列表 → 徽章/块标注更新。

> `range` 交给后端参数（阶段 2 已在 scanner 层实现 `filterByRange`），前端不再二次过滤。

## 4. 组件契约（要点）

- **SourceTabs**：`value`/`onChange`；两个 Tab 显示名（可附 `GET /api/sources` 的计数，复用阶段 3 自检）。键盘可达、选中态青色。
- **RangeFilter**：三段 `7天/1月/全部` → `'7d'|'30d'|'all'`。
- **SessionList**：入参 `sessions/loading/error/selectedId/onSelect`；用 `group.ts` 得 `{day, items[]}[]`，按日期倒序；每组可折叠（默认展开今天/最近）；条目 = 标题（Lora）+ 来源/轮数/时间 + `ExportedBadge`；入场 stagger 30–40ms、悬停 `translateY(-2px)`、选中青色描边。
- **ExportedBadge**：`status:'none'|'partial'|'full'`；full=● partial=◐（青色柔光，不闪）；`none` 不渲染；`title` 提示（如「已导出 2 块」）。同时有图标+文字，颜色非唯一信息载体（docs/04 §6）。
- **ConversationReader**：入参 `meta/rounds/selected/onToggle/thinking/onThinkingChange`；头部显示标题/来源/轮数/时间 + 思考开关；正文按轮渲染「提问/回复」两块，每块左侧 checkbox + 块级已导出标注；空 answer 显示占位。代码块用 `font-mono` + 浅底。可滚动区。
- **ExportBar**：入参 `filename/onFilenameChange/aiTidy/onAiTidyChange/selectedCount/onExport/exporting/lastPath`；无勾选时禁用导出；AI 开关旁提示「预留功能，当前按原样导出」（docs/06 §4）；导出中禁用并显示进度；成功显示路径。
- **ui/ 小组件**：`tabs`(按钮组)、`checkbox`(原生 input + 青色选中)、`switch`、`input`、`badge`——**手写最小实现**（沿用阶段 3，不跑 shadcn CLI、不引额外 Radix 依赖），token 用晨雾变量。

## 6. 动效与可达性（docs/04 §3/§6/§7）

- 列表入场：进入即渲染，逐条 `fadein + translateY` stagger 30–40ms（用 index*delay；reduced 时关）。
- 悬停：条目/按钮 `translateY(-2px)` + 阴影渐显 + 底色转 `--muted`，时长 180–280ms。
- 选中态：青色细描边，无位移抖动；checkbox 勾选打钩淡入。
- 全站已有 `:focus-visible` 青色环、reduced-motion 全局降级（阶段 3 落地）；本期新增动效都走 CSS transition/animation，自动被降级覆盖。
- 点击区 ≥32px；颜色非唯一信息载体（徽章带符号+文字）；对比度达标。

## 7. 接口对接（已就绪，阶段 2）

| 用途 | 调用 |
|------|------|
| 分类计数 | `GET /api/sources` →（已用于顶栏） |
| 列表 | `GET /api/sessions?source=&range=` → `Session[]` |
| 分轮 | `GET /api/sessions/:id?thinking=` → `{meta, rounds}` |
| 导出 | `POST /api/export` `{id, blockIds[], filename?, aiTidy}` → `{ok, path}` |

- `lib/api.ts` 统一：非 2xx → 抛 `{status,message}`；网络失败 → 离线错误（已有）。各组件按 loading/error/empty 三态渲染。
- `blockIds` 用块自身 id（`r{n}-q`/`r{n}-a`，来自 `Round.question/answer.id`）。

## 9. 风险与对策

| 风险 | 对策 |
|------|------|
| 长对话渲染卡顿 | 本期不做虚拟化；右栏滚动容器 + 合理分块即可；量极大留后续迭代 |
| 导出后状态不同步 | 导出成功统一 `refetch` 当前会话 + 当前列表，靠后端 manifest 真值，不做乐观假设 |
| 切会话残留勾选 | `selectedId` 变化时清空 `selected` 与 filename |
| 手写 ui 组件可达性 | checkbox 用原生 input（键盘/读屏天然）；Tab/Switch 加 `role`/`aria-selected`/`aria-checked` |
| Preview 截图 networkidle 超时（字体） | 沿用阶段 3：用 `preview_inspect/eval` 核对 DOM/计算样式 + 必要时单张截图 |
