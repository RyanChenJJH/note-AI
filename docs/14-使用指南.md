# 14 · 使用指南

> 面向使用者（含未来的你自己）。**只讲怎么用**，不讲怎么实现。
> 实现细节按需翻 `docs/02–13`；接力交接看 `docs/00`。
> 最后更新：2026-06-01（阶段 6 完工，v0 可用）

## 0. 这工具是干嘛的（30 秒版）

你跟 Claude Code / Codex CLI 聊过的对话，本地都以 `.jsonl` 文件存着。这工具：
1. 扫出本机所有会话，按来源 + 日期分组列出来；
2. 让你点开某次对话**按轮挑选**要哪几段（提问/回复独立勾选）；
3. 一键导出成**干净 Markdown** + YAML front matter，存进你的 Obsidian 仓库；
4. 已经导出过的会用 ● / ◐ 徽章标出来，不会重复导出心智成本。

源数据**只读**，不会写回 `.claude`/`.codex`。只往**导出目录**和 `server/data/manifest.json` 写。

---

## 1. 第一次使用（一次性准备）

### 1.1 装 Node.js
- 需要 **Node.js ≥ 24**
- 命令行 `node -v` 验证；显示 `v24.x.x` 即可
- 没装：去 https://nodejs.org 下 LTS

### 1.2 装依赖（项目目录）

打开 PowerShell 进项目根目录，分别在 `server/` 和 `web/` 下跑一次 `npm install`：

```powershell
cd server
npm install
cd ..\web
npm install
cd ..
```

> 这步只需要做一次；之后双击 `start.cmd` 即可。
> 如果你后来更新了项目代码（`git pull` 或同步），看到 `package.json` 有变化就再跑一次 `npm install`。

### 1.3 确认导出目录

默认导出到 `E:\Work2\Obsidians_database\00_资源库\02_AI对话记录`。

- 想用这个目录：保证它存在（不存在第一次导出时会自动建）；
- 想改路径：编辑 `server/data/config.json`（首次跑后才会生成）里的 `exportDir`；或临时用环境变量 `AIDA_EXPORT_DIR`（见 §6）。

---

## 2. 日常启动

**双击项目根目录的 `start.cmd`**。它会：

1. 检查 `web/dist`：
   - **不存在** → 自动跑 `npm run build`（约 10–30 秒）；
   - **已存在** → 直接跳过，秒开。
2. 起后端服务（最小化的命令行窗口，标题 `AIDA · API`）。
3. 探测 `/api/sources` 就绪后（最多等 ~20 秒），自动用默认浏览器打开 `http://127.0.0.1:8787`。

启动后看到**封面页**（呼吸光晕 + 「进入」按钮）→ 点进入 → 两栏主界面。

> **关掉工具**：关闭最小化的 "AIDA · API" 窗口即可。
> 主 `start.cmd` 窗口可以直接关掉，不影响已起的后端。

## 3. 主界面操作流程

```
┌─────────────────────────┬─────────────────────────────────┐
│  左栏：会话清单           │  右栏：完整对话                   │
│  ─────────────────       │  ───────────────────              │
│  [Claude] [Codex]        │  [思考过程开关]                   │
│  [7d] [30d] [全部]       │                                   │
│                          │  ## 第 1 轮 · 提问                │
│  ▼ 2026-05-30            │  ☑ ...内容...                    │
│    ● 给我写一段...          │                                  │
│    ◐ 帮我看下错误...        │  ## 第 1 轮 · 回复               │
│                          │  ☑ ...内容...                    │
│  ▼ 2026-05-29            │                                   │
│    ○ 重构这个函数...        │  ## 第 2 轮 · 提问               │
│                          │  ☐ ...内容...                    │
│                          │  ─────────────────                │
│                          │  [文件名______]  [AI 整理 ⏼]      │
│                          │  [导出为 .md]                     │
└─────────────────────────┴─────────────────────────────────┘
```

### 3.1 选源（左栏顶部）

`[Claude] [Codex]` 切换。**点了哪个才加载哪个**（懒加载，省内存）。

- Claude：`%USERPROFILE%\.claude\projects\**\*.jsonl`
- Codex：`%USERPROFILE%\.codex\sessions\YYYY\MM\DD\rollout-*.jsonl`

### 3.2 选范围

`[7d] [30d] [全部]`：按会话起始时间过滤。默认 30d。

### 3.3 浏览列表

- 按日期分组、可折叠（点日期切换展开）；
- 每条标题前的徽章：
  - **●** 完整导出（该会话所有轮的提问 + 回复都导出过）
  - **◐** 部分导出（至少一个块导出过，但不全）
  - **无** 未导出
- 标题来自该会话**第一条真人提问**（截断到 40 字）。

### 3.4 读对话（右栏）

点列表里的标题 → 右栏渲染完整对话。

- 按**轮**分块：「第 N 轮 · 提问」「第 N 轮 · 回复」；
- 每块独立 checkbox；
- 已导出过的块右上角标个小角标（你不会再次勾上不知情）；
- 顶部「思考过程」开关：默认关（去掉 Claude 的 `thinking` 块 / Codex 的 `<think>` 推理）；打开后回复会变长。

### 3.5 勾选 + 命名 + 导出（底部 ExportBar）

1. 勾你要的块（可跨轮，可只勾提问/只勾回复）；
2. **文件名**：留空 → 后端用标题前 20 字；想自定义就填（无须 `.md` 后缀，自动加；非法字符自动替换；重名自动追加序号）；
3. **AI 整理开关**：本期**仅占位**，开着也只是原样拼接 + 显示降级提示「AI 整理为预留功能，已按原样导出」；
4. 点 **导出为 .md** → 写入导出目录 → 列表徽章 / 块级标注立即刷新。

导出成功的提示里会带**完整路径**，方便你直接复制去 Obsidian 验。

## 4. 导出文件长什么样

```markdown
---
source: claude
sessionId: 1a2b3c4d-...
title: "给我写一段Python..."
project: "E:\\Work2\\xxx"
startedAt: 2026-05-30T10:23:11+08:00
exportedAt: 2026-06-01T18:53:00+08:00
rounds: [1, 3]
---

## 第 1 轮 · 提问

给我写一段 Python，把目录 A 里所有 ...

## 第 1 轮 · 回复

可以用 `pathlib` 配合 ...

## 第 3 轮 · 回复

...（如果你只勾了第 3 轮的回复，第 2 轮就不会出现）...
```

- **front matter**：来源、原会话 id（用于追溯）、项目路径、起始/导出时间、本次导出涉及哪几轮；
- **正文**：你勾选的块按轮升序排，提问在回复前；
- 字符编码 UTF-8，换行 `\n`。

## 5. 已导出索引（manifest）在哪

文件：`server/data/manifest.json`

记的是「某会话被导出过哪些块、何时、到哪个 .md」。**只本地用**，不要手动改。

如果你想"重新来过"：删了这个文件即可（列表上的徽章会全部回到「无」状态，下次再导出就重建）。

## 6. 自定义（可选环境变量）

启动前在 PowerShell 里设环境变量，再起 `start.cmd`：

```powershell
$env:AIDA_PORT = "9000"                                # API 端口
$env:AIDA_EXPORT_DIR = "D:\笔记\对话"                   # 临时改导出目录（不动 config.json）
$env:AIDA_STATIC_DIR = "C:\path\to\custom-dist"        # 换前端静态根（高级用法）
$env:AIDA_MANIFEST = "D:\test-manifest.json"           # 隔离 manifest（测试用）
$env:AIDA_CONFIG = "D:\test-config.json"               # 隔离配置（测试用）
.\start.cmd
```

- 这些变量只对当前 PowerShell 窗口有效；关掉就失效。
- 想永久改导出目录：编辑 `server/data/config.json` 里的 `exportDir`。
- 改了 `AIDA_PORT` 后浏览器会被打开到旧地址（脚本里写死 8787）；手动改 URL 即可，或编辑 `start.cmd` 里两处端口。

## 7. 改前端后想看到效果

`start.cmd` 默认**只在 web/dist 不存在时构建**。如果你改了 `web/src/` 下的代码：

**方法 A · 重新构建（推荐日常）**
```powershell
Remove-Item -Recurse -Force web\dist
.\start.cmd
```

**方法 B · 开发模式（实时热更新）**
两个 PowerShell 窗口分别跑：
```powershell
# 窗口 1
cd server
npm run serve
# 窗口 2
cd web
npm run dev
```
然后浏览器开 http://localhost:5173（Vite dev，自动代理 `/api → 8787`）。

## 8. 常见问题

### Q1. 双击 `start.cmd` 没反应/一闪而过
打开 PowerShell 进项目根，手动跑一遍看报错：
```powershell
.\start.cmd
```
最常见原因：
- 没装 Node 或版本 < 24
- `server/` 或 `web/` 下没 `npm install` 过

### Q2. 浏览器开了但页面打不开 / `ERR_CONNECTION_REFUSED`
- 看「AIDA · API」窗口里的日志，多半是端口被占用
- `netstat -ano | findstr 8787` 看占用进程，杀掉或换 `AIDA_PORT`

### Q3. 列表是空的
- 确认你确实用过 Claude Code / Codex CLI（在本机至少跑过一次会话）
- 切「全部」范围看是不是被时间筛掉了
- 切到另一个源 Tab 看另一边有没有
- 极端情况：`%USERPROFILE%\.claude\projects` 或 `%USERPROFILE%\.codex\sessions` 路径异常 → 用 `AIDA_CLAUDE_ROOT` / `AIDA_CODEX_ROOT` 显式覆盖

### Q4. 标题出现乱码 / 内容截断
- 标题是从第一条**真人提问**截到 40 字，长就是会截
- 乱码：看 jsonl 原文件是不是损坏；本工具按 UTF-8 读

### Q5. 导出失败 / "无法写入"
- 导出目录不存在且无权限创建：手动建一下，或把 `exportDir` 改到一个能写的位置
- 文件名含非法字符（`< > : " / \ | ? *`）：会自动替换为 `_`，不至于失败；如果还是失败，看后端窗口报错

### Q6. AI 整理开关勾上了为什么还是原样？
本期**只占位**。开关打开会调一个降级实现（原样拼接），并在导出成功后显示「AI 整理为预留功能，已按原样导出」。真实接入三 provider 是后续迭代（见 `docs/06 §5`）。

### Q7. 导出过的对话还想再导一次
- 直接再勾再导就行；新文件会自动避免重名（追加 `-2`、`-3`…）
- 想让徽章重置：删 `server/data/manifest.json`

### Q8. 思考过程开关勾上后回复变得很长
正常的。Claude 的 `thinking` 块与 Codex 的 `<think>...</think>` 推理默认折叠，开关打开后会展开拼入回复正文。一般用不上，调试模型行为时再开。

### Q9. 我能把这个工具搬到 Mac / Linux 用吗
后端是纯 Node，前端是纯 web，理论上都能跑。但是：
- `start.cmd` 是 Windows 批处理，不能直接用 → 你需要写个等价 `start.sh`（构建 + serve + 开浏览器）
- 默认导出路径用了 Windows 反斜杠；要改成你系统的路径
- 数据源路径默认 `~/.claude/projects` 与 `~/.codex/sessions`，跨平台一致，无需改

---

## 9. 想关闭/卸载

- **临时停服**：关掉 "AIDA · API" 窗口。
- **彻底清空状态**：删 `server/data/manifest.json` 与 `server/data/config.json`（不删导出的 .md）。
- **卸载**：删整个项目文件夹。源 `.claude` / `.codex` 是 IDE/CLI 自己的，不归这工具管。
- 已导出的 .md 笔记是你自己的，独立存活，不依赖本工具。
