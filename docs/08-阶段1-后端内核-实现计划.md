## 2. 实测发现（文档之外、已影响设计的真实细节）

> 2026-06-01 读本机真实会话核对所得，是下面设计的依据。上游格式会演进，解析需容错。

### 2.1 Claude 侧
- 实际见到的 `type`：`permission-mode`、`file-history-snapshot`、`user`、`attachment`(skill_listing)、`assistant`、`system`(subtype `turn_duration`)。除 `user`/`assistant` 外全部丢弃。
- ⚠️ **嵌套子代理文件**：`projects\<proj>\<uuid>\subagents\agent-*.jsonl` 是 sidechain 旁路。
  - scanner **必须排除**（路径含 `subagents` 段的不当作独立会话）。
  - parser 另按记录的 `isSidechain:true` 丢弃。
- ⚠️ **思考混入正文**：部分 `assistant` 的 `text` 块（自定义 provider 场景）把推理写进正文并以 `</think>` 收尾，而非独立 `thinking` 块。
  - 处理：`includeThinking=false`（默认）时，正文若含 `</think>`，裁掉 `</think>` 及其之前的部分。

### 2.2 Codex 侧
- 真人首问确认位置：`response_item` + `payload.type:"message"` + `role:"user"` + `content:[{type:"input_text", text}]`。
- 首问前的噪音顺序：`developer`(permissions/collaboration/apps/skills/plugins) → `user(<environment_context>)` → `turn_context` → `event_msg(task_started)`。
- ⚠️ **重复记录**：同一条消息还会以 `event_msg`(`user_message` / `agent_message`) 再出现一次。
  - **只从 `response_item`+`message` 取文本，彻底忽略所有 `event_msg`**，否则每轮内容翻倍。
- `session_meta` 的 cwd 在 `payload.cwd`；`payload.timestamp` 是 **UTC**(`...Z`)；而**目录 `YYYY/MM/DD` 与文件名时间是本地时间**（实测差 +08:00）。→ 见 §7 日期分组决策。

## 5. 模块职责与要点

### 5.1 `jsonl.ts` — 共享读取底座
- 以 UTF-8 读整文件，按 `/\r?\n/` 切行，逐行 `try { JSON.parse } catch { console.warn; continue }`。
- **绝不抛错**；返回成功解析的对象数组（或生成器）。
- 文件较小（本机单会话 KB~MB 级，共约 134 个），整读即可，后续需要再换流式。

### 5.2 `scanner.ts` — 会话清单（不返回正文）
- 遍历目录用 `fs.readdir(dir,{recursive:true})`。
  - Claude：`~/.claude/projects/*/*.jsonl`，**跳过路径段含 `subagents` 的文件**。
  - Codex：`~/.codex/sessions/YYYY/MM/DD/rollout-*.jsonl`。
- 每个文件做**轻量解析**（复用 parser 的消息抽取，只为拿首问与计数）：
  - `title` = 首条真人提问截断（≤40 字，超长加省略号）。
  - `turnCount` = 真人提问条数（= 分轮数）。
  - `day` / `startedAt` / `project(cwd)` 见 §6、§7。
- `exported` 状态由 `manifest.deriveSessionStatus` 注入（none/partial/full）。
- 空会话（无真人提问）：标记后列表可隐藏。
- `range`(7d/30d/all) 参数预留，阶段 1 默认 all（筛选逻辑阶段 2 接入）。

### 5.3 `parser/` — 核心：过滤 + 归一 + 分轮
- `claude.ts` / `codex.ts`：各按 `docs/03` 过滤规则，把原始记录归一成 `NormalizedMessage`，并抽会话级 meta。
- `rounds.ts`：纯函数，吃 `NormalizedMessage[]`，跑 `docs/03 §C`：**先过滤 → 按 timestamp 排序 → 遇真人提问开新轮，其后 AI 文字回复并入当前轮**。同轮多条 assistant（工具循环）全部并入。
- `index.ts`：`resolveSessionPath(id)` 按 `${source}:${uuid}` 用 uuid 在两源 glob 定位文件；`parseSession` 分派 → 返回 `{meta, rounds}`。
- `includeThinking` 开关贯穿：默认 false 丢 thinking（含正文里 `</think>` 之前的部分）；true 保留。

### 5.4 `exporter.ts` — 选中块 → Markdown
- 按 `docs/05 §3`：YAML front matter（title/source/session_id/project/date/exported_at/rounds/tags）+ 正文。
- 只含勾选块；每块 `## 第 N 轮 · 提问/回复` 小标题；正文原样（保留代码块）；块按轮升序、提问在回复前。
- 文件名净化：非法字符 `\ / : * ? " < > |` → `-`；自动补 `.md`；同名追加 ` (2)`/` (3)`。
- 导出目录不存在自动创建。

### 5.5 `manifest.ts` — 已导出索引（docs/05 §4）
- 存 `server/data/manifest.json`，结构见 docs/05 §4.2；只增量追加，绝不写 `.claude`/`.codex`。
- `recordExport(id, {file, blocks, aiTidy})` 追加一条 export。
- `deriveSessionStatus(id, allBlockIds)`：汇总该会话所有 `exports[].blocks` → none/partial/full。
- `deriveBlockStatus(id)`：返回已导出块 id 集合，供块级标注。

### 5.6 `cli.ts` — 自测入口（`node:util.parseArgs`）
| 命令 | 作用 |
|------|------|
| `scan [--source claude\|codex] [--json]` | 打印会话清单（表格或 JSON） |
| `show <id> [--thinking]` | 打印某会话分轮结果（验证解析） |
| `export <id> --blocks r1-q,r1-a [--filename x]` | 导出选中块为 md + 记 manifest |
| `verify <id>` | §F 自检：轮数 / 被过滤噪音条数 / 标题取值 / thinking 开关差异 |

## 6. 数据结构（`types.ts`）

```ts
type Source = 'claude' | 'codex';

interface Session {
  id: string;          // `${source}:${sessionId}`，全局唯一
  source: Source;
  title: string;       // 首条真人提问（截断）
  startedAt: string;   // ISO 时间
  day: string;         // YYYY-MM-DD，用于日期分组（见 §7）
  project?: string;    // 记录里的 cwd
  turnCount: number;   // 真人提问轮数
  exported: 'none' | 'partial' | 'full';
}

interface Round {
  index: number;       // 第几轮，从 1 开始
  question: Block;     // 人类提问
  answer: Block;       // AI 文字回复（多段合并；可能为空）
}

interface Block {
  id: string;          // `r${index}-q` | `r${index}-a`
  text: string;
  exported: boolean;   // 来自 manifest
}

// parser 内部归一化中间态
interface NormalizedMessage {
  type: 'question' | 'answer';
  text: string;
  ts: string;          // ISO，用于排序
  thinking?: boolean;  // 该段是否思考内容
}

interface SessionMeta {
  id: string;
  source: Source;
  sessionId: string;
  title: string;
  startedAt: string;
  day: string;
  project?: string;
}
```

manifest 结构见 `docs/05 §4.2`。

## 7. 已锁定决策（本阶段新增）

1. **日期分组时区**：`day` 优先取**本地日期**。
   - Codex：用目录路径 `YYYY/MM/DD`（本地，权威）。
   - Claude：仅有 UTC 的 `Z` 时间戳，取其日期（无本地信息可用）。
   - 理由：避免跨午夜会话归错天。
2. **阶段 1 导出目录**：CLI 测试导到 `server/data/_exports_test/`，**不污染** Obsidian 库；正式默认值 `E:\Work2\Obsidians_database\00_资源库\02_AI对话记录` 仍写入 `paths.ts` 作为生产默认（阶段 2+ 用）。

## 9. 硬约束（贯穿全程）

- **只读源数据**：scanner/parser 绝不写 `.claude`/`.codex`。
- **只写 `server/`**：md 导出目录 + `data/manifest.json`。
- **容错**：遇未知 `type`/`payload.type` 跳过，不报错；单行解析失败跳过并 warning。
- **编码**：统一 UTF-8。
- **路径来自记录内部**：项目 cwd 取记录里的 `cwd`/`session_meta.payload.cwd`，不反解目录名。
