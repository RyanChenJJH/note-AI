## 2. 文件命名

| 模式 | 规则 |
|------|------|
| 手动（默认） | 用户在 ExportBar 填写文件名；缺省值可用"标题前 N 字-日期" |
| AI 介入开启后 | 自动命名（如 `{日期}-{AI生成的主题}.md`，见 `docs/06`） |

- 自动补 `.md` 后缀；非法字符（`\ / : * ? " < > |`）替换为 `-`。
- 同名已存在：追加 ` (2)`、` (3)` 或提示用户改名（二选一，建议提示）。

## 3. Markdown 格式

带 Obsidian 友好的 YAML front matter + 正文。**只包含勾选的块**。

```markdown
---
title: "你可以做什么工作"
source: claude            # claude | codex
session_id: "92a529cc-...-7d45bdfe4167"
project: "C:/Users/RyanC/Desktop/test"
date: 2026-05-31          # 会话日期
exported_at: 2026-06-01T08:50:00+08:00
rounds: [1, 3]            # 本文件涉及的轮次
tags: [AI对话, claude]
---

## 第 1 轮 · 提问

你可以做什么工作

## 第 1 轮 · 回复

（AI 文字回复正文……）

## 第 3 轮 · 回复

（AI 文字回复正文……）
```

格式约定：
- 每个勾选块一个 `## 第 N 轮 · 提问/回复` 小标题，正文原样保留（含代码块）。
- 块顺序按轮次升序、提问在回复前。
- front matter 的 `rounds` 标明涉及哪些轮，便于回查。
- 标签、标题层级等可在后续做成可配置模板。

## 4. 已导出追踪（manifest）

### 4.1 存储

- 一个 JSON：`server/data/manifest.json`（与工具一起，独立于源数据与导出目录）。
- 只增量更新；绝不写入 `.claude`/`.codex`。

### 4.2 结构

```jsonc
{
  "version": 1,
  "records": {
    // key = `${source}:${sessionId}`
    "claude:92a529cc-...": {
      "title": "你可以做什么工作",
      "exports": [
        {
          "exportedAt": "2026-06-01T08:50:00+08:00",
          "file": "E:/Work2/Obsidians_database/00_资源库/02_AI对话记录/xxx.md",
          "blocks": ["r1-q", "r1-a", "r3-a"],   // 导出了哪些块
          "aiTidy": false
          // 阶段7：手动导出记 "file"(单篇)；AI 整理记 "files": [多个路径] 且 aiTidy:true
        }
      ]
    }
  }
}
```

### 4.3 状态推导（给 UI）

对某会话，汇总它所有 `exports[].blocks`：
- 没有任何导出 → `exported: "none"`
- 导出过部分块 → `exported: "partial"`（◐）
- 覆盖了全部块（该会话所有轮的 q/a 都被导出过）→ `exported: "full"`（●）

块级：某块 id 出现在任一 `exports[].blocks` 里 → 该块 `exported: true`，UI 标注 + 悬停显示导出时间/文件。

### 4.4 交互建议

- 已导出**不阻止**再次导出（用户可能要重导/合并）；但导出前若检测到块已导出，给一个轻提示（toast，可忽略）。
- 提供"打开所在文件夹 / 打开该 md"的快捷入口（可选）。

## 5. 边界

- 导出文件被用户手动删除/移动：manifest 仍保留记录（路径可能失效）；可加"校验文件是否存在"的弱提示，不强删记录。
- 同一会话多次导出不同块：在 `exports[]` 里追加多条，状态推导自动合并。
- 跨机器/重装：manifest 可随项目一起版本管理或备份。
