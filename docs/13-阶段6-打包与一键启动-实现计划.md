## 3. 静态托管设计

### 3.1 路由优先级

```text
请求进来 → matchRoute(routes, method, pathname)
  ├─ 命中 API 路由 → 按既有逻辑处理
  └─ 未命中
       ├─ method ∈ {GET, HEAD} 且 staticDir 存在 →
       │     ├─ 尝试 `<staticDir>/<pathname>`（normalize 后必须仍在 staticDir 内）
       │     │     ├─ 存在且是文件 → 200 + 正确 MIME
       │     │     └─ 不存在/是目录
       │     │           ├─ pathname 以 `/api/` 开头 → 404（不污染 API 命名空间）
       │     │           └─ 否则 → 回 `<staticDir>/index.html`（SPA fallback）
       │     └─ index.html 也不存在 → 404
       └─ 其它（POST 等 + 未命中） → 既有 404/405 逻辑
```

- `pathMatched` 仍按 router 现有语义返回（路径模板匹配但方法错 → 405），**优先于**静态层。即 `/api/sessions` 用 PUT 仍 405，不会被静态层接走。
- 静态层缺失（`AIDA_STATIC_DIR` 指向不存在的目录、或默认 `web/dist` 还没构建）→ 直接走原 404，**dev 期照常用 Vite 代理**不受影响。

### 3.2 路径安全（避免目录穿越）

```ts
// static.ts 内
const root = resolve(dir);
const target = resolve(root, '.' + decodeURIComponent(pathname));
if (!target.startsWith(root + sep) && target !== root) return null; // 拒绝
```

- 用 `path.resolve` + 边界检查，拒掉 `../` 与符号链接逃逸。
- 仅暴露 `staticDir` 内文件；不读 `staticDir` 之外任何路径。

### 3.3 MIME

最小够用表（其它一律 `application/octet-stream`）：

| 后缀 | MIME |
|------|------|
| `.html` | `text/html; charset=utf-8` |
| `.js` / `.mjs` | `text/javascript; charset=utf-8` |
| `.css` | `text/css; charset=utf-8` |
| `.json` | `application/json; charset=utf-8` |
| `.svg` | `image/svg+xml` |
| `.png` / `.jpg`/`.jpeg` / `.gif` / `.webp` | 对应 image/* |
| `.woff` / `.woff2` | `font/woff(2)` |
| `.ico` | `image/x-icon` |
| `.txt` / `.map` | `text/plain; charset=utf-8` |

- 不设 `Cache-Control`（本地小工具，每次刷新拿最新即可）；不做 gzip/ETag。
- `index.html` SPA fallback 用 200（不是 302）；不缓存。

### 3.4 同源 & CORS

- 代码现状：`server/src` 全文检索无 `Access-Control-*` 头（已确认），**无需删除任何 CORS**。
- 生产：浏览器与 API 同源（`http://127.0.0.1:8787`），不需要 CORS。
- 开发：Vite dev 仍代理 `/api → 8787`，端口同变量（见 `vite.config.ts`），保持开发体验不变。

## 5. `server.ts` 接入

```ts
// 伪代码
const deps = { ..., staticDir: overrides.staticDir ?? staticDir() };

createServer(async (req, res) => {
  const url = new URL(req.url ?? '/', 'http://127.0.0.1');
  const { match, pathMatched } = matchRoute(routes, req.method ?? 'GET', url.pathname);

  if (match) { /* 既有：handler + try/catch */ return; }
  if (pathMatched) { sendError(res, 405, '方法不被允许'); return; }

  // 静态层兜底（仅 GET/HEAD；/api/* 不走 SPA fallback）
  if ((req.method === 'GET' || req.method === 'HEAD') && deps.staticDir) {
    const served = await tryServeStatic(req, res, deps.staticDir, url.pathname);
    if (served) return;
  }
  sendError(res, 404, '未找到路由');
});
```

- `tryServeStatic` 返回 `boolean`（已写响应=true，未处理=false）。
- 仅在 `/api/*` 之外做 SPA fallback；`/api/unknown` 永远走 404 JSON（保留客户端错误语义）。
- 错误（读文件失败等）→ 走既有 `try/catch` 500 兜底；进程不崩。

## 6. `start.cmd`（项目根）

```bat
@echo off
setlocal
cd /d "%~dp0"

echo [AIDA] 检查前端构建产物...
if not exist "web\dist\index.html" (
  echo [AIDA] 未发现 web\dist，开始构建（首次较慢，约 10-30 秒）...
  pushd web
  call npm run build || (echo [AIDA] 前端构建失败 & popd & pause & exit /b 1)
  popd
) else (
  echo [AIDA] 已存在 web\dist，跳过构建（如需刷新前端请删除该目录或手动 npm run build）。
)

echo [AIDA] 启动本地服务 http://127.0.0.1:8787 ...
start "" /min cmd /c "cd /d server && npm run serve"

REM 等服务起来再开浏览器（最多 ~10 秒）
set /a tries=0
:waitloop
powershell -NoProfile -Command "try { (Invoke-WebRequest -UseBasicParsing -TimeoutSec 1 http://127.0.0.1:8787/api/sources).StatusCode } catch { exit 1 }" >nul 2>&1
if %errorlevel%==0 goto ready
set /a tries+=1
if %tries% GEQ 20 goto giveup
timeout /t 1 /nobreak >nul
goto waitloop

:ready
echo [AIDA] 服务已就绪，打开浏览器...
start "" http://127.0.0.1:8787
goto end

:giveup
echo [AIDA] 等待服务超时（10s），仍尝试打开浏览器...
start "" http://127.0.0.1:8787

:end
endlocal
```

- **缺 dist 才 build**：判断 `web\dist\index.html` 存在与否，符合用户选择「若 web/dist 不存在才 build」。
- **后端独立窗口最小化运行**：`start "" /min cmd /c "..."`，关掉 start.cmd 主窗口不会拖走服务；用户改完代码可手动关那个最小化窗口重启。
- **就绪探测**：`/api/sources` 200 即就绪；最多等 ~10s 后兜底直接开。
- 端口固定 8787（与 `serverPort()` 默认一致；如需改，README 说明改环境变量后再起）。
