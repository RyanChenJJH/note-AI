## 2. skill 集成契约（实测自 SKILL.md）

**输入**：对话 md 全文（必给，含 AIDA frontmatter `title/source/date/session_id/project` + `## 第 N 轮 · 提问/回复`）；`source_id`（可选，来源路径）；`date`（可选，整理日期，默认今天）。

**输出**：哨兵分隔块，**原文不转义**，除块外无任何其它文字：
```
<<<NOTE filename="20260602-概念-MCP原理.md">>>
---
标题: 概念-MCP原理
类型: 概念
领域: 技术沉淀
...（笔记自带的中文 frontmatter）
---
（正文，含任意代码/引号/换行）
<<<END NOTE>>>
```
- 多篇连续输出多个块，块间空一行。
- 无可沉淀知识 → 仅一行 `<<<NO NOTES: 原因>>>`。
- ⚠️ **每篇笔记自带 frontmatter**，AIDA **不再为其套对话 frontmatter**。

## 3. 关键变更：`tidy()` 契约 单篇 → 多篇

阶段 5 的 `TidyResult.markdown: string`（单段正文，由 exporter 套 frontmatter，写 1 文件）**装不下**真实 skill。新契约：

```ts
// server/src/ai/tidy.ts
export interface TidyNote { filename: string; content: string; } // content 含各自 frontmatter
export interface TidyInput {
  conversationMd: string;   // 勾选块拼成的完整对话 md（buildMarkdown 产物，含 AIDA frontmatter）
  meta: SessionMeta;        // 供 source_id/来源 拼接、日期兜底
  date: string;             // 整理日期 YYYY-MM-DD（默认今天）
}
export interface TidyResult {
  usedAi: boolean;          // 是否真走了模型
  notes: TidyNote[];        // 0+ 篇；degraded 时为空
  noNotes?: string;         // 模型判定无可沉淀知识时的原因（<<<NO NOTES: ...>>>）
  degraded?: string;        // 降级原因（未配置/超时/鉴权失败/网络/解析失败）；有值时 notes 为空、usedAi=false
}
export async function tidy(input: TidyInput, cfg: AiConfig): Promise<TidyResult>;
```

降级语义：`degraded` 有值 ⇒ AI 不可用，前端提示用户「关掉 AI 开关按原样导出」（走手动路径），**预览阶段不自动写盘**。

## 5. 后端：AI 管线模块（全部收敛在 `server/src/ai/`）

### 5.1 `ai/skill.ts` — 载入并拼 system prompt
```ts
export interface LoadedSkill { systemPrompt: string; root: string; }
export function loadSkill(skillPath: string): LoadedSkill;
```
- **定位 SKILL.md**：`skillPath` 直接含 `SKILL.md` 则用之；否则向下浅搜（≤3 层）取最浅的 `SKILL.md`（兼容用户给父目录 `my_personal_skill` 的情况）。
- 读 `SKILL.md`（去掉它给 Claude Code 用的 YAML frontmatter，留正文）+ `references/type-selection.md` + `references/templates/*.md`（全部，按文件名排序）。
- 用清晰分隔拼成 system prompt：
  ```
  你是「AI 对话 → 知识笔记」整理助手。严格遵循下面 SKILL 指令工作。
  【最重要·输出契约】只输出 <<<NOTE filename="...">>> … <<<END NOTE>>> 块（多篇连排），
  无可沉淀知识则只输出一行 <<<NO NOTES: 原因>>>；不要任何前言/解释/代码围栏/转义。
  ===== SKILL.md =====
  <正文>
  ===== references/type-selection.md =====
  <...>
  ===== references/templates/概念页.md =====
  <...>  …（全部模板）
  ```
- 读不到 SKILL.md → 抛错，由 `tidy()` 捕获转 `degraded: "未找到 skill（路径：…）"`。
- 规模约 750 行 ≈ 12–18K tokens，DeepSeek 64K 上下文充裕。读盘很小，每次现读即可（可选按 mtime 缓存，非必需）。

### 5.2 `ai/provider.ts` — provider 无关的对话接口
```ts
export interface ChatMessage { role: 'system' | 'user' | 'assistant'; content: string; }
export interface ChatProvider { chat(messages: ChatMessage[], cfg: AiConfig, signal: AbortSignal): Promise<string>; }
export function getProvider(p: AiProvider): ChatProvider;
```
- **v1 实现 `openai-compatible`**（`ai/providers/openai.ts`）：`POST {baseUrl}/chat/completions`，`Authorization: Bearer {apiKey}`，body `{ model, messages, temperature, stream:false }`，取 `choices[0].message.content`。非 2xx → 抛带状态码的错误（401/429/5xx 文案）。
- `anthropic` / `ollama`：留同形 stub，抛「provider 暂未实现」（后续迭代填，见 [docs/06 §5](06-AI整理接口设计.md)）。

### 5.3 `ai/parse.ts` — 哨兵块解析
```ts
export interface ParseResult { notes: TidyNote[]; noNotes?: string; }
export function parseNotes(raw: string): ParseResult;
```
- 正则/扫描提取所有 `<<<NOTE filename="(...)">>>\n(正文)\n<<<END NOTE>>>`；容错：filename 缺失 → 兜底名；尾随/前导空白、外层误包代码围栏 → 剥掉再解析。
- 命中 `<<<NO NOTES: (原因)>>>` 且无 NOTE 块 → `{ notes: [], noNotes }`。
- 一个 NOTE 块都解析不到且无 NO NOTES → 返回空 + 让 `tidy()` 视为 `degraded: "模型输出无法解析"`（把原始输出记日志便于排查）。

### 5.4 `ai/tidy.ts` — 编排（替换占位实现）
```
if (!cfg.enabled) return degraded("未开启 AI 整理")
if (provider 需要 key 且 cfg.apiKey 为空) return degraded("未配置 API Key")
try:
  skill = loadSkill(cfg.skillPath)
  messages = [ {system: skill.systemPrompt}, {user: 组装用户消息(input)} ]
  raw = await getProvider(cfg.provider).chat(messages, cfg, timeoutSignal(cfg.timeoutMs))
  { notes, noNotes } = parseNotes(raw)
  if (!notes.length && !noNotes) return degraded("模型输出无法解析")
  return { usedAi:true, notes, noNotes }
catch e:
  return degraded( e=AbortError? "AI 超时" : 鉴权/网络/未实现 文案 )
```
- 用户消息 = 对话 md 全文 + 一行提示（`整理日期: {date}`；若有 source_id 则附）。
- 超时：`AbortController` + `setTimeout(cfg.timeoutMs)`。
- 任何异常都转 `degraded`，**绝不抛到路由**（沿用「进程不崩、可降级」原则）。

## 6. 后端：路由（预览 / 提交 / 手动）

`server.ts` 路由表新增两条；现有 `/api/export` 保持不变（仅默认目录随 config 改到 00_收件箱）。

| 方法 路径 | 入参 | 出参 | 说明 |
|---|---|---|---|
| POST `/api/ai/preview` | `{ id, blockIds[] }` | `{ usedAi, notes:[{filename,content}], noNotes?, degraded? }` | 跑 `tidy()`，**不写盘** |
| POST `/api/ai/commit` | `{ id, blockIds[], notes:[{filename,content}] }` | `{ ok, paths:string[] }` | 把确认后的 notes 写入 **aiNotesDir**，记 manifest |
| POST `/api/export`（现有） | `{ id, blockIds[], filename? }` | `{ ok, path, usedAi:false }` | 手动原样单文件 → **exportDir(00_收件箱)**；不再读 aiTidy |

- `ai/preview.ts`：resolve 会话 → `parseSessionFile` → `buildMarkdown(meta,rounds,blockIds)` 作 `conversationMd` → `tidy()` → 原样返回（含 degraded/noNotes）。
- `ai/commit.ts`：把客户端确认（可能改过文件名/取消了某些篇）的 `notes` 逐篇 `writeExport(content, aiNotesDir, filename)`；**一条** manifest 记录覆盖本次 `blockIds`（`files=所有写出路径`、`aiTidy=true`）。本地单人工具，commit 直接写客户端回传内容（不二次跑模型，避免不确定性与重复成本）。
- 手动 `export.ts`：移除 `aiTidy`/`tidy()` 分支，回到纯 `buildMarkdown` + `writeExport` 到 `exportDir`。

### 与蓝图的差异（追溯）
1. **路由放平**：用 `routes/ai-preview.ts` / `routes/ai-commit.ts`（而非蓝图 §6 的 `routes/ai/` 子目录），与现有 `routes/*.ts` 同层；并抽 `routes/_session.ts` 去掉 export/preview/commit 的重复样板。
